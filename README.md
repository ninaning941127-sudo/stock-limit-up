# 台股漲停策略回測

回測一個很單純的假設性策略：**只要股票當天觸及漲停，就假設能買到漲停價進場**，再依當天收盤時的狀態分兩種出場方式：

- **鎖死到收盤**（收盤價 = 最高價 = 漲停價）：隔天開盤立刻賣出
  `pnl = 隔天開盤價 - 當天收盤價`
- **觸及漲停但沒鎖死**（收盤價 < 最高價）：當天收盤賣出
  `pnl = 當天收盤價 - 當天最高價`

漲停價以「前一天收盤價 × 1.10」估算（留 0.1% 容忍誤差處理跳動點位取整）。

## 檔案

| 檔案 | 說明 |
|---|---|
| `limit_up_backtest.py` | 單一個股回測（預設聯發科 2454.TW，資料來自 yfinance），2026 年初至今 |
| `market_limit_up_backtest.py` | 全台股回測，讀取 TEJ 匯出的 Excel，涵蓋 2026 年初至今約 1,900 檔股票 |
| `trades.csv` / `market_trades.csv` | 個股／全市場的交易明細 |
| `market_daily_ic.csv` | 每日 IC（鎖死訊號 vs 實際報酬率）時間序列 |
| `*.png` | 各項圖表（累積損益/報酬率曲線、報酬率分布、每日 IC） |

## 執行方式

```bash
pip install yfinance pandas numpy matplotlib seaborn scipy openpyxl

python limit_up_backtest.py            # 單一個股（2454.TW）
python market_limit_up_backtest.py     # 全台股（需先把 market_limit_up_backtest.py
                                        # 裡的 DATA_XLSX 指到你的 TEJ 匯出檔路徑）
```

`market_limit_up_backtest.py` 會輸出：
- 勝率、平均報酬率等基本統計（整體 / 鎖死 / 未鎖死分開列）
- t 檢定（單樣本、兩樣本 Welch）與線性迴歸 R²（檢驗累積報酬率趨勢是否顯著）
- 每日 IC / IR 分析（鎖死訊號對實際報酬率的橫斷面 Spearman 相關係數）

## 目前結果的重點結論

以 2026 年上半年資料為例（6,301 筆漲停事件、1,276 檔股票）：

- 整體勝率 55.2%，平均每筆報酬率 +1.35%
- 拆開來看：鎖死（4,575 筆）勝率 76%、平均 +3.33%；未鎖死（1,726 筆）勝率**必然是 0%**（因為出場價=收盤價、進場價=當天最高價，收盤不可能高於當天最高，屬於策略定義下的數學必然，不是發現）
- 所有統計檢定（t 檢定、迴歸 p 值）都顯示上述效應在統計上顯著，IC/IR 分析也顯示鎖死與未鎖死的報酬率差異每天都很穩定（IR ≈ 3.7）

## 已知限制（表現好看≠可實際獲利，要注意）

1. **鎖死的獲利幾乎不可能真的買到**：股票會鎖死，正是因為當天買單遠多於賣單；回測假設「只要漲停就買得到」，但實際上這正是最難成交的情境。獲利完全來自這個最不容易成交的族群，是目前回測最大的高估來源。
2. **沒有計入交易成本**：證交稅（賣出 0.3%）、手續費都沒扣。
3. **未考慮資金與同時發生的部位限制**：同一天常有數十檔同時漲停，回測把每筆交易當獨立事件加總報酬率，並未反映真實本金只有一份、必須分配给同時出現的訊號。
4. **R² 接近 1 主要是累加序列的統計特性**（只要平均值不為 0，累加曲線本來就會趨近直線），不代表策略每天風險很低。
5. 樣本僅半年，且台股當時小型股投機氣氛濃厚，換個市場環境結果可能不同。

下一步如果要驗證這個策略「真的能不能賺」，最關鍵的是拿到實際成交量/委買委賣資料，把「鎖死但你其實排不到單、買不到」的情境排除掉，看看剩下的報酬還剩多少。
