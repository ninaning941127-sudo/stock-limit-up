# 台股漲停策略回測

回測一個很單純的假設性策略：**只要股票當天觸及漲停，就假設能買到漲停價進場**，再依當天收盤時的狀態分兩種出場方式：

- **鎖死到收盤**（收盤價 = 最高價 = 漲停價）：隔天開盤立刻賣出
  `pnl = 隔天開盤價 - 當天收盤價`
- **觸及漲停但沒鎖死**（收盤價 < 最高價）：當天收盤賣出
  `pnl = 當天收盤價 - 當天最高價`

漲停價的算法：前一天收盤價 × 1.10，再**無條件捨去到該價位的 TWSE 升降單位（tick）**（例如 100–500 元 tick=0.5、≥1000 元 tick=5），得到當天實際漲停價，再用精確比對判斷是否觸及／鎖死。

## 檔案

| 檔案 | 說明 |
|---|---|
| `limit_up_backtest.py` | 單一個股回測（預設聯發科 2454.TW，資料來自 yfinance），2026 年初至今 |
| `market_limit_up_backtest.py` | 全台股回測，讀取 TEJ 匯出的 Excel，涵蓋 2026 年初至今約 1,900 檔股票 |
| `market_liquidity_backtest.py` | 以成交量當流動性 proxy，估算「漲停鎖死」的名目獲利現實中有多少比例真的買得到 |
| `trades.csv` / `market_trades.csv` | 個股／全市場的交易明細 |
| `market_liquidity_trades.csv` | 全市場交易明細（含成交量、量比、可成交比例、名目/折算後損益等流動性欄位） |
| `market_daily_ic.csv` | 每日 IC（鎖死訊號 vs 實際報酬率）時間序列 |
| `*.png` | 各項圖表（累積損益/報酬率曲線、報酬率分布、每日 IC、流動性四分位/散佈/折算曲線） |

## 執行方式

```bash
pip install yfinance pandas numpy matplotlib seaborn scipy openpyxl

python limit_up_backtest.py            # 單一個股（2454.TW）
python market_limit_up_backtest.py     # 全台股（需先把腳本裡的 DATA_XLSX
                                        # 指到你的 TEJ 匯出檔路徑）
python market_liquidity_backtest.py    # 流動性分析（同樣需先設定自己的 DATA_XLSX，
                                        # 且它預設指向另一個含成交量欄位的 TEJ 匯出檔）
```

`market_limit_up_backtest.py` 會輸出：
- 勝率、平均報酬率等基本統計（整體 / 鎖死 / 未鎖死分開列）
- **按交易日聚合後的 t 檢定（主要結論）**：同一天數十檔同時漲停的報酬高度相關，逐筆當獨立樣本會嚴重高估顯著性，故以每日平均報酬、交易日數為有效樣本數檢定；逐筆版本僅保留為對照
- 每日 IC / IR 分析（鎖死訊號對實際報酬率的橫斷面 Spearman 相關係數）——**這是對群聚結構最穩健的主要證據**
- 累積報酬率曲線的線性迴歸 R² 僅為描述性（累加序列本來就近似直線，非策略穩定度）

`market_liquidity_backtest.py` 會輸出：名目 vs 流動性折算後的總損益與剩餘比例、鎖死組相對量比／可成交比例與報酬率的相關性、依相對量比分位的報酬率、以及對應圖表。

## 目前結果的重點結論

以 2026 年上半年資料為例，改用 tick 取整的正確漲停價算法後（10,061 筆漲停事件、1,423 檔股票——比舊算法多抓到約 60% 的真實漲停事件，主要是之前被漏掉的高價股）：

- 整體勝率 54.1%，平均每筆報酬率 +1.22%
- 拆開來看：鎖死（7,250 筆）勝率 75.1%、平均 +3.22%；未鎖死（2,811 筆）勝率**必然是 0%**（因為出場價=收盤價、進場價=當天最高價，收盤不可能高於當天最高，屬於策略定義下的數學必然，不是發現）
- **按交易日聚合後的 t 檢定**（主要結論）：鎖死組每日平均報酬 +3.07%（120 個交易日，t=17.9，p≈1.2e-35）、未鎖死組每日平均 -3.88%（121 天，t=-36.1，p≈3.0e-66），效應在按日聚合、排除群聚高估之後依然高度顯著
- IC/IR 分析顯示鎖死與未鎖死的報酬率差異每天都很穩定（平均 IC=0.657，IR≈4.2），這是對群聚結構最穩健的證據

## 已知限制（表現好看≠可實際獲利，要注意）

1. **鎖死的獲利幾乎不可能真的買到**：股票會鎖死，正是因為當天買單遠多於賣單；回測假設「只要漲停就買得到」，但實際上這正是最難成交的情境。獲利完全來自這個最不容易成交的族群，是目前回測最大的高估來源。
2. **沒有計入交易成本**：證交稅（賣出 0.3%）、手續費都沒扣。
3. **未考慮資金與同時發生的部位限制**：同一天常有數十檔同時漲停，回測把每筆交易當獨立事件加總報酬率，並未反映真實本金只有一份、必須分配给同時出現的訊號。
4. **R² 接近 1 主要是累加序列的統計特性**（只要平均值不為 0，累加曲線本來就會趨近直線），不代表策略每天風險很低。
5. **除息／除權／減資日的參考價調整未處理**（本次修正範圍外）：漲停價用未還原的前收盤 ×1.1 計算，但這些日子交易所的參考價會調整——除息日真實漲停價較低會**漏抓**真漲停，減資／併股日參考價較高則可能把正常漲幅**誤判**為漲停。要根除需另外接除權息日期表或還原股價。
6. 樣本僅半年，且台股當時小型股投機氣氛濃厚，換個市場環境結果可能不同。

下一步如果要驗證這個策略「真的能不能賺」，最關鍵的是拿到實際成交量/委買委賣資料，把「鎖死但你其實排不到單、買不到」的情境排除掉，看看剩下的報酬還剩多少。
